Git Checkout entre Commits

O comando git checkout pode ser usado para navegar entre commits. Quando você usa git checkout HEAD, você volta para o commit mais recente da branch atual. Não há mudanças visíveis se você já estiver no estado mais recente, mas pode ser útil se você tiver feito alterações não commitadas e quiser restaurar o diretório de trabalho ao último commit.

Voltar para um Commit Anterior Relativo:

Com o comando git checkout HEAD~N, você retorna ao estado do repositório de N commits atrás, onde N é o número de commits que você deseja voltar.
git checkout HEAD~1 volta um commit atrás.
git checkout HEAD~2 volta dois commits atrás, e assim por diante.

Checkout para um Commit Específico

Para voltar a um commit específico, use o código do commit com git checkout.
Por exemplo: git checkout 6dcb09bcca3eabcd123456789abcde123456789a
