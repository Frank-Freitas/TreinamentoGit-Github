Comandos no git bash

branch

Uma branch é como uma linha do tempo paralela do seu projeto. Quando você cria uma branch, está criando uma nova linha de desenvolvimento que começa a partir de um ponto específico na história do projeto, mas que pode seguir seu próprio caminho de forma independente da linha principal, sem alterar a mesma.

ls

Lista todos os arquivos e diretórios no diretório atual.

ls -1

Mostra uma lista detalhada com mais informações.

ls -a

Lista todos os arquivos incluindo arquivos ocultos.

ls -1a

Combina os dois comandos -1 e -a mostrando uma lista detallhada com todos os arquivos ocultos.

git init

Inicializa um novo repositório dentro do diretório em que você deu o comando, significa que foi criado uma pasta va oculta e vazia ".git".

git status

Mostrar informações importantes do que esta acontecendo em seu repositório do git.

git add.

Adicionar mudanças nos arquivos na area de preparação. Significa que estamos dizendo ao git que desejamos incluir essas novas alterações no proximo commit.

git add algo especifico

usando o git add para mandar apenas uma parte de todas as modificações que foram feitas

git commit -m

É utilizado para salvar as atualizações que você adicionou ao repositório com o comando git add. E ao usar git commit -m "nome da atualização", você fornece uma descrição para essas atualizações(um nome), que será registrada no histórico da branch. A branch representa a linha do tempo do projeto,mostrando todos os passos da vida da quele projeto.

git branch -M main

Renomear uma branch para o nome main. o M significa renomar/ alterar o nome. Se uma branch com o nome de destino (neste caso, main) já existir, o -M força a substituição dessa branch existente. Isso é útil se você deseja garantir que a branch seja renomeada independentemente de qualquer conflito de nomes.

git push

É usado para enviar as alterações locais feitas no seu repositório Git para um repositório remoto (github). Quando você faz um git push, você está atualizando o repositório remoto com os commits que você fez localmente.

git log

Exibe o histórico de commits do repositório Git. Ele mostra uma lista de commits recentes, detalhando informações sobre cada um.

git log --oneline

Mostrar o histórico de versões de modo simplificado, com apenas o inicio do codigo e o nome do commit.

git reset

Depois de ter dado um git add . e usar um git reset, remove todos os arquivos que foram colocados na area de staged,, com o git add . Tambem existe outras maneiras de implementar mais coisas no git reset

git diff

O git diff mostra a diferença entre a versão modificada e a anterior de um codigo especifico quando voce colocar no git bash "git diff nome-do-arquivo.html".

git checkout

O comando git checkout pode ser usado para varias operações, como mudar de brabch, restauraar arquivos espécificos e navegar entre os commits. Ver mais sobre checkout

Deletando Ultimo commit sem deletar modificações nos arquivos.

Sempre usando o git status para ver o status atual.
Localmente: O comando git reset --soft HEAD~1 move o ponteiro HEAD para HEAD~1, ou seja, o penúltimo commit agora se torna o commit "atual" no seu repositório local. O commit que era o HEAD mais recente é "desfeito" localmente e suas mudanças são colocadas na área de stage. No histórico local: Agora, HEAD~1 se torna HEAD, e o commit que era HEAD ainda existe, mas suas alterações estão prontas para serem re-commitadas, modificadas ou removidas.

No Repositório Remoto:

Nada muda imediatamente: O repositório remoto ainda tem HEAD e HEAD~1 como estavam antes do reset local. O commit que você "desfez" localmente ainda está lá como o commit mais recente no remoto. Após git push --force (se você fizer isso): Se você decidir executar git push --force após o git reset --soft HEAD~1, você estará sobrescrevendo o histórico no remoto com o histórico local atualizado. Resultado no remoto: O commit que era o HEAD~1 agora se tornará o HEAD no remoto, e o commit original do HEAD será removido do histórico remoto (ele será perdido, a menos que esteja referenciado em outro lugar).
