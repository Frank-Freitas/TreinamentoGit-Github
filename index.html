Entre no repositório do GitHub e pegue a chave SSH do repositório. Em seguida, abra o Git Bash na pasta do seu projeto e use o comando git remote add origin, seguido do link do seu repositório no GitHub. O comando ficará assim: git remote add origin git@github.com:seuusuario/seurepositorio.git

Comandos no Git Bash

Branch

Uma branch é como uma linha do tempo paralela do seu projeto. Quando você cria uma branch, está criando uma nova linha de desenvolvimento que começa a partir de um ponto específico na história do projeto, mas pode seguir seu próprio caminho de forma independente da linha principal, sem alterar a principal.

ls

Lista todos os arquivos e diretórios no diretório atual.

ls -l

Mostra uma lista detalhada com mais informações sobre os arquivos e diretórios.

ls -a

Lista todos os arquivos, incluindo arquivos ocultos.

ls -la

Combina as opções -l e -a, mostrando uma lista detalhada com todos os arquivos, incluindo os ocultos.

git init

Inicializa um novo repositório no diretório atual. Isso cria uma pasta oculta chamada ".git".

git status

Mostra informações sobre o estado atual do seu repositório Git.

git add .

Adiciona todas as mudanças na área de preparação (staging area). Isso indica ao Git que você deseja incluir essas alterações no próximo commit.

git add algo específico

Usa o comando git add para adicionar apenas uma parte das modificações feitas.
Ver mais

git commit -m

Salva as atualizações adicionadas ao repositório com o comando git add. Ao usar git commit -m "mensagem", você fornece uma descrição para essas atualizações, que será registrada no histórico da branch. A branch representa a linha do tempo do projeto, mostrando todos os passos do projeto.

git branch -M main

Renomeia a branch atual para "main". O -M força a substituição de uma branch existente com o mesmo nome, se houver. Isso é útil para garantir que a branch seja renomeada, independentemente de conflitos de nomes.

git push

Envia as alterações locais feitas no seu repositório Git para um repositório remoto (como o GitHub). Com git push, você atualiza o repositório remoto com os commits que fez localmente.

code .

Abre o VS Code a partir do Git Bash, na pasta em que você está quando executa o comando code .

git log

Exibe o histórico de commits do repositório Git, mostrando uma lista de commits recentes e detalhando informações sobre cada um.

git log --oneline

Mostra o histórico de commits de forma simplificada, com apenas o início do código e a mensagem do commit.

git reset

Após usar git add ., o comando git reset remove todos os arquivos da área de preparação (staging area). Existem outras maneiras de usar o git reset para diferentes fins.

git diff

Mostra a diferença entre a versão modificada e a versão anterior de um arquivo específico. Use git diff nome-do-arquivo.html para ver as alterações.

git checkout

O comando git checkout pode ser usado para várias operações, como mudar de branch, restaurar arquivos específicos e navegar entre commits.

Deletar o último commit sem remover as modificações nos arquivos

Sempre use git status para verificar o estado atual.
Localmente: O comando git reset --soft HEAD~1 move o ponteiro HEAD para o commit anterior (HEAD~1). O commit mais recente é "desfeito" localmente e suas mudanças são colocadas na área de preparação. No histórico local, HEAD~1 se torna o novo HEAD, e o commit que era HEAD ainda existe, mas suas alterações estão prontas para serem re-commitadas, modificadas ou removidas.

No Repositório Remoto:

Nada muda imediatamente: O repositório remoto ainda tem HEAD e HEAD~1 como estavam antes do reset local. O commit que você "desfez" localmente ainda está lá como o commit mais recente no remoto.
Após usar git push --force (se você decidir fazer isso): Se você executar git push --force após o git reset --soft HEAD~1, você estará sobrescrevendo o histórico remoto com o histórico local atualizado.
Resultado no remoto: O commit que era HEAD~1 se tornará o novo HEAD no remoto, e o commit original de HEAD será removido do histórico remoto (ele será perdido, a menos que esteja referenciado em outro lugar).

git pull

O comando git pull é essencial para manter seu repositório local atualizado com as mudanças feitas no repositório remoto. Ele busca e mescla as mudanças de um branch específico do repositório remoto para o branch local atual.

O que pode acontecer:

Mesclagem automática: Se não houver conflitos entre as mudanças locais e as mudanças no remoto, a mesclagem é feita automaticamente e as alterações são aplicadas no seu branch local.
Conflitos: Se houver conflitos entre as mudanças locais e as mudanças do remoto, o Git irá parar e solicitar que você resolva os conflitos manualmente antes de concluir a mesclagem.
Atualizações de branch: Se o branch remoto tiver novos commits que não estão no branch local, git pull aplicará esses commits ao seu branch local.

Resolver push rejeitado por histórico.

Se o repositório local estiver atrasado em relação ao remoto, o git push normalmente falhará. Isso acontece porque o Git detecta que o repositório remoto tem commits que não estão no seu repositório local. O Git faz isso para evitar sobrescrever mudanças feitas por outras pessoas.

Para resolver esse problema, você deve primeiro atualizar seu repositório local com as alterações do remoto usando git pull. Isso trará os commits do remoto para o seu repositório local. Se o comando git pull for executado com sucesso e não houver conflitos, você poderá fazer o git push normalmente.

Se houver conflitos durante o git pull, você precisará resolvê-los antes de conseguir fazer o git push.

Após o git pull, se for necessário fazer um merge, o editor padrão (como o VIM) pode abrir para você inserir uma mensagem de commit para o merge. Aqui está como você pode usar o VIM:

Como usar o VIM para inserir uma mensagem de commit:

  1. Quando o VIM abrir, você estará no modo de comando. Para começar a editar, pressione i para entrar no modo de inserção.
  2. Digite a mensagem de commit. Normalmente, o Git já sugere uma mensagem de commit para merges.
  3. Após inserir sua mensagem, pressione Esc para voltar ao modo de comando.
  4. Para salvar e sair, digite :wq e pressione Enter. Isso escreve a mensagem e sai do VIM.
  5. Se você quiser sair sem salvar, pressione :q! e depois Enter.

Com a mensagem de commit salva, o Git continuará o processo de merge e você poderá fazer o git push para atualizar o repositório remoto com suas mudanças.
