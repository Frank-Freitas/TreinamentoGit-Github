Aprendendo a usar o Git bash

Comandos no git bash

branch

Uma branch é como uma linha do tempo paralela do seu projeto. Quando você cria uma branch, está criando uma nova linha de desenvolvimento que começa a partir de um ponto específico na história do projeto, mas que pode seguir seu próprio caminho de forma independente da linha principal, sem alterar a mesma.

ls

Lista todos os arquivos e diretórios no diretório atual.

ls -1

Mostra uma lista detalhada com mais informações.

ls -a

Lista todos os arquivos incluindo arquivos ocultos.

ls -1a

Combina os dois comandos -1 e -a mostrando uma lista detallhada com todos os arquivos ocultos.

git init

Inicializa um novo repositório dentro do diretório em que você deu o comando, significa que foi criado uma pasta va oculta e vazia ".git".

git status

Mostrar informações importantes do que esta acontecendo em seu repositório do git.

git -add.

Adicionar mudanças nos arquivos na area de preparação. Significa que estamos dizendo ao git que desejamos incluir essas novas alterações no proximo commit.

git commit -m

É utilizado para salvar as atualizações que você adicionou ao repositório com o comando git add. E ao usar git commit -m "nome da atualização", você fornece uma descrição para essas atualizações(um nome), que será registrada no histórico da branch. A branch representa a linha do tempo do projeto,mostrando todos os passos da vida da quele projeto.

git branch -M main

Renomear uma branch para o nome main. o M significa renomar/ alterar o nome. Se uma branch com o nome de destino (neste caso, main) já existir, o -M força a substituição dessa branch existente. Isso é útil se você deseja garantir que a branch seja renomeada independentemente de qualquer conflito de nomes.

git push

É usado para enviar as alterações locais feitas no seu repositório Git para um repositório remoto (github). Quando você faz um git push, você está atualizando o repositório remoto com os commits que você fez localmente.
